# CLI Mode Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 163MusicLyrics 添加一个独立的命令行（CLI）项目，复用现有业务逻辑，支持按歌曲/专辑/歌单 ID 下载歌词到本地文件。

**Architecture:** 新建 `MusicLyricApp.CLI` console 项目，通过 `<ProjectReference>` 直接引用现有的 `MusicLyricApp` 项目（与测试项目做法相同），从而复用 `SearchService`、`LyricUtils` 等全部业务逻辑，无需移动任何文件。CLI 入口通过 `System.CommandLine` 解析参数，直接调用 `SearchService.SearchSongs()` 并将结果写入文件，完全绕过 Avalonia 及所有 ViewModel。

**Tech Stack:** .NET 9.0 · System.CommandLine 2.0-beta4 · NLog（复用）· 现有 MusicLyricApp 服务层

---

## 可行性说明

**完全可行**，原因：

1. **业务逻辑已高度解耦** —— `SearchService`、`LyricUtils`、所有 Music API 均不依赖 Avalonia。
2. **测试项目先例** —— `MusicLyricApp.Tests` 已通过 `<ProjectReference>` 引用 `MusicLyricApp`（WinExe），证明该方式可行。
3. **关键入口点明确**：
   - `SearchService.SearchSongs(List<InputSongId>, SettingBean)` — 返回 `Dictionary<string, ResultVo<SaveVo>>`
   - `LyricUtils.GetOutputContent(LyricVo, SettingBean)` — 返回 `List<string>`，每个元素是一个完整文件的内容（通常只有 1 个，`SeparateFileForIsolated=true` 时会有多个）
   - `GlobalUtils.CheckInputId(string, SearchSourceEnum, SearchTypeEnum)` — 解析 ID/URL
   - `GlobalUtils.GetOutputName(SaveVo, string, string, int)` — 生成输出文件名

## CLI 用法设计

```
用法: 163music-cli <IDs> [选项]

参数:
  <IDs>                    歌曲/专辑/歌单 ID 或 URL，多个用逗号分隔

选项:
  --source/-s <netease|qq>             音乐来源 (默认: netease)
  --type/-t <song|album|playlist>      搜索类型 (默认: song)
  --format/-f <lrc|srt>                输出格式 (默认: lrc)
  --output/-o <目录>                    输出目录 (默认: 当前目录)
  --lrc-type/-l <stagger|isolated|merged>  歌词合并方式 (默认: stagger)
  -h, --help                           显示帮助

示例:
  163music-cli 123456
  163music-cli 123456 --type album --output ./lyrics
  163music-cli 123456,234567 --source qq --format srt
  163music-cli "https://music.163.com/song?id=123456"
```

### Task 2: 编写 CliRunner 核心逻辑

**Files:**
- Create: `cross-platform/MusicLyricApp.CLI/CliRunner.cs`

> **关键说明：**
> - `LyricUtils.GetOutputContent` 返回 `List<string>`，**每个元素是一个完整文件的内容**（不是行列表），需要逐元素写文件
> - 文件名使用 `GlobalUtils.GetOutputName` 生成，与 GUI 行为一致
> - 此类不涉及任何 Avalonia 或 ViewModel

- [ ] **Step 1: 创建 `CliRunner.cs`**

```csharp
// cross-platform/MusicLyricApp.CLI/CliRunner.cs
using System;
using System.Collections.Generic;
using System.IO;
using System.Threading.Tasks;
using MusicLyricApp.Core.Service;
using MusicLyricApp.Core.Utils;
using MusicLyricApp.Models;

namespace MusicLyricApp.CLI;

public static class CliRunner
{
    public static async Task<int> RunAsync(
        string[] ids,
        SearchSourceEnum source,
        SearchTypeEnum type,
        OutputFormatEnum format,
        ShowLrcTypeEnum lrcType,
        string outputDir)
    {
        // 1. 构建 SettingBean（使用默认值，后续可扩展为从 config 加载）
        var setting = new SettingBean();
        setting.Param.SearchSource = source;
        setting.Param.OutputFileFormat = format;
        setting.Param.ShowLrcType = lrcType;

        // 2. 构建 SearchService
        var searchService = new SearchService(setting);

        // 3. 展开 IDs（song 直接用，album/playlist 需查询展开）
        var inputSongIds = new List<InputSongId>();
        foreach (var rawId in ids)
        {
            try
            {
                // CheckInputId 解析 ID 或 URL，返回含 QueryId 的对象（SongId 未设置）
                var parsed = GlobalUtils.CheckInputId(rawId.Trim(), source, type);
                var musicApi = searchService.GetMusicApi(parsed.SearchSource);

                switch (parsed.SearchType)
                {
                    case SearchTypeEnum.SONG_ID:
                        // 2-arg 构造：SongId = 第一个参数，QueryId 继承自 parsed
                        inputSongIds.Add(new InputSongId(parsed.QueryId, parsed));
                        break;
                    case SearchTypeEnum.ALBUM_ID:
                        var albumVo = musicApi.GetAlbumVo(parsed.QueryId).Assert().Data;
                        foreach (var song in albumVo.SimpleSongVos)
                            inputSongIds.Add(new InputSongId(song.DisplayId, parsed));
                        break;
                    case SearchTypeEnum.PLAYLIST_ID:
                        var playlistVo = musicApi.GetPlaylistVo(parsed.QueryId).Assert().Data;
                        foreach (var song in playlistVo.SimpleSongVos)
                            inputSongIds.Add(new InputSongId(song.DisplayId, parsed));
                        break;
                }
            }
            catch (Exception ex)
            {
                Console.Error.WriteLine($"[ERROR] 解析 ID 失败: {rawId} — {ex.Message}");
                return 1;
            }
        }

        if (inputSongIds.Count == 0)
        {
            Console.Error.WriteLine("[ERROR] 没有有效的歌曲 ID");
            return 1;
        }

        // 4. 搜索歌词
        Console.WriteLine($"正在下载 {inputSongIds.Count} 首歌的歌词...");
        var results = searchService.SearchSongs(inputSongIds, setting);

        // 5. 保存结果（参照 StorageService.WriteToFile 的逻辑）
        Directory.CreateDirectory(outputDir);
        var ext = setting.Param.OutputFileFormat.ToDescription().ToLower();
        var encoding = GlobalUtils.GetEncoding(setting.Param.Encoding);
        var successCount = 0;

        foreach (var (songId, resultVo) in results)
        {
            if (!resultVo.IsSuccess())
            {
                Console.Error.WriteLine($"[WARN] 歌曲 {songId} 下载失败: {resultVo.ErrorMsg}");
                continue;
            }

            var saveVo = resultVo.Data;

            if (saveVo.LyricVo.IsEmpty())
            {
                Console.Error.WriteLine($"[WARN] 歌曲 {songId}（{saveVo.SongVo.Name}）暂无歌词");
                continue;
            }

            // GetOutputContent 返回 List<string>：每个元素是一个完整文件的内容字符串
            // isSingle=true 时只有 1 个文件；SeparateFileForIsolated=true 时可能有多个
            var res = await LyricUtils.GetOutputContent(saveVo.LyricVo, setting);
            var baseName = GlobalUtils.GetOutputName(
                saveVo,
                setting.Config.OutputFileNameFormat,
                setting.Config.SingerSeparator,
                setting.Config.SingerCountLimit);
            var isSingle = res.Count == 1;

            for (var i = 0; i < res.Count; i++)
            {
                var fileName = isSingle ? $"{baseName}.{ext}" : $"{baseName}-{i}.{ext}";
                var filePath = Path.Combine(outputDir, fileName);
                await File.WriteAllTextAsync(filePath, res[i], encoding);
                Console.WriteLine($"[OK] {fileName}");
            }

            successCount++;
        }

        Console.WriteLine($"\n完成：{successCount}/{results.Count} 首歌词已保存到 {Path.GetFullPath(outputDir)}");
        return successCount == results.Count ? 0 : 1;
    }
}
```

```csharp
// cross-platform/MusicLyricApp.CLI/Program.cs
using System.CommandLine;
using MusicLyricApp.CLI;
using MusicLyricApp.Models;

var idsArg = new Argument<string[]>(
    name: "ids",
    description: "歌曲/专辑/歌单 ID 或 URL，多个用逗号分隔")
{
    Arity = ArgumentArity.OneOrMore
};

var sourceOpt = new Option<string>(
    aliases: ["--source", "-s"],
    description: "音乐来源: netease (默认) | qq",
    getDefaultValue: () => "netease");

var typeOpt = new Option<string>(
    aliases: ["--type", "-t"],
    description: "搜索类型: song (默认) | album | playlist",
    getDefaultValue: () => "song");

var formatOpt = new Option<string>(
    aliases: ["--format", "-f"],
    description: "输出格式: lrc (默认) | srt",
    getDefaultValue: () => "lrc");

var outputOpt = new Option<string>(
    aliases: ["--output", "-o"],
    description: "输出目录（默认：当前目录）",
    getDefaultValue: () => ".");

var lrcTypeOpt = new Option<string>(
    aliases: ["--lrc-type", "-l"],
    description: "歌词合并方式: stagger (默认) | isolated | merged",
    getDefaultValue: () => "stagger");

var rootCommand = new RootCommand("163MusicLyrics CLI — 下载网易云/QQ音乐歌词");
rootCommand.AddArgument(idsArg);
rootCommand.AddOption(sourceOpt);
rootCommand.AddOption(typeOpt);
rootCommand.AddOption(formatOpt);
rootCommand.AddOption(outputOpt);
rootCommand.AddOption(lrcTypeOpt);

rootCommand.SetHandler(async (ids, source, type, format, output, lrcType) =>
{
    var searchSource = source.ToLower() switch
    {
        "qq" => SearchSourceEnum.QQ_MUSIC,
        _ => SearchSourceEnum.NET_EASE_MUSIC
    };

    var searchType = type.ToLower() switch
    {
        "album" => SearchTypeEnum.ALBUM_ID,
        "playlist" => SearchTypeEnum.PLAYLIST_ID,
        _ => SearchTypeEnum.SONG_ID
    };

    var outputFormat = format.ToLower() switch
    {
        "srt" => OutputFormatEnum.SRT,
        _ => OutputFormatEnum.LRC
    };

    var lrcTypeEnum = lrcType.ToLower() switch
    {
        "isolated" => ShowLrcTypeEnum.ISOLATED,
        "merged" => ShowLrcTypeEnum.MERGE,
        _ => ShowLrcTypeEnum.STAGGER
    };

    // 支持逗号分隔的多 ID（也支持 shell 传多参数）
    var allIds = new System.Collections.Generic.List<string>();
    foreach (var id in ids)
        allIds.AddRange(id.Split(',', System.StringSplitOptions.RemoveEmptyEntries));

    var exitCode = await CliRunner.RunAsync(
        [..allIds], searchSource, searchType, outputFormat, lrcTypeEnum, output);

    System.Environment.Exit(exitCode);
},
idsArg, sourceOpt, typeOpt, formatOpt, outputOpt, lrcTypeOpt);

return await rootCommand.InvokeAsync(args);
```

> **注意：** `ShowLrcTypeEnum.ISOLATED`、`ShowLrcTypeEnum.MERGE`、`OutputFormatEnum.LRC/SRT` 的枚举名称需与 `SettingBase.cs` 中的实际定义核对。如有出入按实际名称修改。

## 注意事项

1. **Cookie 问题：** 网易云音乐和 QQ 音乐 API 可能需要 Cookie 才能获取完整歌词。如果下载失败，先在 GUI 版本中登录一次并保存设置，然后使用 Task 4 的方式加载配置；或直接在 `CliRunner.cs` 中设置 `setting.Config.NetEaseCookie = "..."` / `setting.Config.QQMusicCookie = "..."`。

2. **Avalonia DLL 会出现在输出目录：** 由于引用了 `MusicLyricApp` 项目，Avalonia 相关 DLL 会被复制到 CLI 的输出目录。但 CLI 代码从不调用任何 Avalonia 代码，这些 DLL **不会被初始化**，不需要 X11/Wayland 环境。发布为 `PublishSingleFile=true` 时所有内容打包为单一可执行文件。

3. **枚举名称核对：** `ShowLrcTypeEnum`、`OutputFormatEnum` 等枚举的确切值名称需与 `SettingBase.cs` 中的定义一致，编译时会直接报错，按错误提示修改即可。
